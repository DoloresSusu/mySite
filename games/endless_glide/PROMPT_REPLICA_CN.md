# 无尽滑行 (Endless Glide) 项目复刻提示词

## 1. 角色设定
你是一名资深游戏设计师、创意技术专家与前端工程师，擅长使用极简主义美学与程序生成技术，打造具有强烈沉浸感与电影质感的 Web 游戏体验。

## 2. 核心玩法逻辑
- **基础玩法**：横版无尽滑行游戏，角色自动向前推进。
- **操作指令**：
  - `空格键`：跳跃（应用向上瞬时力，配合重力模拟）。
  - `Shift` / `点击` / `触摸`：加速（增加水平速度，配合摩擦力逻辑）。
- **物理特性**：实现平滑的基于惯性的系统。角色速度上限需设得较高（Max Vx: 25+），以营造强烈的速度冲击感。
- **衡量标准**：以“滑行距离（米）”作为核心进度衡量标准。

## 3. 视觉与美学规格 (关键)
- **视觉风格**：极简黑白剪影风格。所有前景元素（角色、地形、植被）必须使用纯黑色。
- **背景系统**：
  - **动态风景**：每滑行 500 米，背景渐变色必须平滑、连续地向下一个色彩阶段（如日落红转深紫、海洋蓝、迷雾灰等）轮换。
  - **氛围营造**：利用 Canvas 渐变与模糊 UI 覆盖层提升画面的高级感。
- **环境演化**：
  - **程序地形**：使用噪声算法（Perlin/Simplex）生成平滑的丘陵，复杂度随距离逐步演化。
  - **视差图层**：实现至少 3 层山峦剪影，不同层具备不同的移动速度以产生纵深感。
  - **装饰元素**：在地表随机生成极简的松树/植物剪影。
- **角色表现**：冲浪者/滑行者剪影。角色身后带有一条黑色固体拟真丝带（Ribbon），随滑行轨迹产生延迟波动效果。

## 4. UI 界面规格
- **游戏内 UI**：
  - 使用加粗、白色的无衬线字体（如 Outfit 或系统默认 Sans）。
  - 左上角显示：“距离: [x] 米”与“速度: [x] 公里/小时”。
  - 底部显示低亮度的操作提示（“空格: 跳跃 | Shift: 加速”）。
- **开始画面**：
  - 全屏沉浸式标题“ENDLESS GLIDE”（大号字体）。
  - 极简中文字样“无尽滑行”。
  - 指示：“按空格或点击开始游戏”。
- **结算画面**：背景高斯模糊，显示最终滑行距离及“重新出发”按钮。

## 5. 技术约束
- **技术栈**：使用 HTML5 Canvas + 原生 JavaScript（不依赖重型引擎）。
- **性能**：使用 `requestAnimationFrame` 确保 60fps 的流畅度。
- **响应式**：强制锁定 12:8 的电影比例，并根据屏幕大小自适应缩放。
- **生成规则**：地形与树木必须实时生成与销毁，确保持续运行不溢出内存。

## 6. 核心哲学
“少即是多”。前景拒绝任何颜色，重点在于运动的流畅性、动作的惯性以及天色变换带来的情绪共鸣。
